# Microsoft Purview DLP Investigation Notes

## Objective

Understand how security analysts investigate Data Loss Prevention (DLP) alerts using Microsoft Purview.

## Investigation Workflow

1. Review Alert Details
2. Identify Sensitive Information Type
3. Validate User Activity
4. Determine Business Justification
5. Assess Risk Level
6. Document Findings
7. Escalate if Required

## Common Sensitive Information Types

- Social Security Numbers
- Payment Card Information
- Financial Data
- Employee Records

## Key Takeaways

- Context matters when reviewing alerts.
- False positives should be documented and tuned.
- DLP policies should balance security and business needs.
