# Insider Risk Investigation Notes

## Objective

Understand how organizations identify, investigate, and respond to insider risk events using Microsoft Purview.

## Key Concepts

### Insider Risk

Insider risk involves user activities that may result in data loss, unauthorized disclosure, or policy violations.

Examples:

- Data exfiltration
- Unauthorized cloud uploads
- Mass file downloads
- Sensitive data sharing

---

## Investigation Workflow

1. Review Alert Information
2. Identify Risk Indicators
3. Analyze User Activities
4. Validate Business Context
5. Determine Risk Severity
6. Document Findings
7. Escalate if Required

---

## Common Risk Indicators

- Large data transfers
- Unusual file access
- External sharing activity
- USB device usage
- Personal cloud uploads

---

## Key Takeaways

- Context is critical when evaluating alerts.
- Not every alert represents malicious activity.
- Proper documentation supports governance and compliance efforts.
- Insider Risk Management complements DLP programs.
